# 轻舟应用开发规范

## 概述

轻舟对业务系统的管控是通过轻舟代理调度对应的“轻舟应用”来完成的。
轻舟应用是基于“轻舟 API”开发的 Java 应用程序。
轻舟 API 为各类业务系统提供统一的开发规范，包括接口、UI、集成方式等，最终实现多品类业务系统的标准化、一致化管理。

## 快速开始

📌 参考示例：`apps/qingzhou-app-demo`

1. 新建项目，在 `qingzhou/apps` 目录下新建 Maven 子模块。
2. 编写应用入口类，入口类会在轻舟启动时被调用。
    - 创建类，实现 `QingzhouApp` 接口。
    - 添加 `@App`、`@Menu` 注解，配置元数据。
    - 重写 `start()` 方法，定制启动逻辑。
3. 编写业务模块类，模块类对应业务实体，在管控台上体现为功能菜单。
    - 创建类，继承 `ModelBase`。
    - 添加 `@Model` 注解，配置元数据。
    - 根据业务需求，实现相应的能力接口，如`List`、`Show`、`Add`、`Update`、`Delete`、`Monitor`等。
4. 添加业务模块字段，模块字段对应业务数据，在管控台上体现为表单字段。
    - 创建属性，如 `public String id`。
    - 添加 `@ModelField` 注解，配置元数据。
5. 添加业务模块操作，模块操作对应 REST 接口，在管控台上体现为按钮。
    - 创建方法，如 `public void myMethod(Request request){}`，注意方法的参数有且只能有一个 Request 类型。
    - 添加 `@ModelAction` 注解，配置元数据。
6. 部署
    - 在新建项目下执行 `mvn clean install` 完成编译。
    - 将生成的 jar 包放入`${轻舟产品包}/instances/default/apps` 目录下。
    - 重启轻舟，访问管控台，可看到开发的应用。

## 图标参考

轻舟 API 支持设置图标，如 `qingzhou.api.App.icon` `qingzhou.api.Menu.icon` `qingzhou.api.Model.icon` 等，
轻舟前端使用 **Element Plus Icons** 图标库渲染页面，因此您可直接使用 Element Plus 官方图标名。可参考：
https://element-plus.org/zh-CN/component/icon.html#icon-collection
