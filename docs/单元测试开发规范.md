# 单元测试开发规范

## 命名规范

- 测试类：`被测类名 + Test`，包名与被测类一致。
- 测试方法：采用 `场景_行为_结果` 的 BDD 风格，如 `nullId_queryUser_throwException`。

## 编写规范

- 自校验：测试结果必须由断言自动判定，避免人工检查日志。
  - 使用 `assertEquals` `assertNotEquals` 断言值
  - 使用 `assertNull` `assertNotNull` 断言空对象
  - 使用 `assertSame`  `assertNotSame` 断言对象引用
  - 使用 `assertTrue` `assertFalse` 断言真假
  - 使用 `fail` 断言其他，
  - 不使用 `logger` `System.out` `System.err` 等日志
- 隔离性：每个测试用例独立运行，不依赖其他用例或执行顺序。
  - 避免使用全局属性或共享变量，不改变公共状态。
- 单一性：一个测试用例只验证一个场景（正向/异常/边界）。
- 专注性：专注在场景验证本身，外部数据库、接口等用模拟数据。
